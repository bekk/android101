--- layout: default title: Consume REST APIs with Retrofit --- Retrofit is a third-party library by Square for communicating with REST APIs over HTTP. Similarly to AsyncTasks, work is done in a background thread, and upon completion it returns the result to the main UI thread. To use Retrofit, declare a service using a simple interface using annotations. Then, use the built-in RestAdapter to create an object which can be used for REST calls. When Retrofit returns the result of the network call to the main UI thread, it automatically maps JSON to Java domain objects when using getters and setters with the same name as the JSON object keys. As Retrofit handles the communication between the main UI thread and a background thread, in addition to JSON parsing, it can greatly simply the workflow by reducing the amount of overhead.

Below is an example of the MessageService.java interface.

{% highlight java %} public interface MessageService { @GET("/messages") void listMessages(Callback> callback); @POST("/message") void send(@Body Message message, Callback callback); } {% endhighlight %} The example below shows usage in an activity that displays the messages. Notice the Callback parameter that returns a success handler and a failure handler. These will be run depending on the success of the request. {% highlight java %} // compile 'com.fatboyindustrial.gson-jodatime-serialisers:gson-jodatime-serialisers:1.0.0' final Gson gson = Converters.registerDateTime(new GsonBuilder()).create(); RestAdapter restAdapter = new RestAdapter.Builder() .baseUrl("http://mobile-course.herokuapp.com") .setConverter(new GsonConverter(gson)) .build(); MessageService msgs = restAdapter.create(MessageService.class); msgs.listMessages(new Callback>() { @Override public void success(List messages, Response response) { // Successfull request, do something with the retrieved messages } @Override public void failure(RetrofitError retrofitError) { // Failed request } }); {% endhighlight %}
